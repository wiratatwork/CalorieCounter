# Calorie Tracker — Test Cases

เอกสารทดสอบด้วยมือ (Manual QA) สำหรับ Wellness Hub + Calorie Counter  
รูปแบบ: **Input → Process → Expected Output**

---

## สิ่งที่ต้องเตรียมก่อนทดสอบ

| รายการ | รายละเอียด |
|--------|------------|
| Environment | เปิดผ่าน local server หรือ GitHub Pages |
| Supabase | ตั้งค่า `js/config.js` แล้ว deploy Edge Function `estimate-calories` |
| Migration | รัน `supabase/migrations/add_meal_tag.sql` (ถ้าต้องการทดสอบแท็กมื้อถาวร) |
| Browser | Chrome / Edge ล่าสุด + จอมือถือ 375px |
| ข้อมูลเริ่มต้น | ล้าง `localStorage` key `calorie_daily_goal` หรือตั้งเป็น `2000` |

## 3. แถบความคืบหน้า vs เป้า (`calorie.html`)

### TC-GOAL-01 แสดง progress หลังบันทึก

| | |

| **Expected Output** | แถบ progress แสดง, เป้า `2,000 kcal`, สถานะ `เหลือ 1,470 kcal`, แถบ ~26% |

### TC-GOAL-03 เกินเป้า (>110%)

## 7. Edge Cases & Error Handling
